# 하드웨어 입력 테스트 배선 가이드

이 문서는 XIAO nRF52840 / XIAO BLE 보드에서 1키, 4키 입력 테스트를 할 때의 배선 기준을 정리한다.

## 전체 여정 위치

```text
1. 기술 검증 → 단일 보드 입력 검증
```

## 기본 원칙

초기 테스트는 가장 단순한 직접 입력 방식으로 진행한다.

```text
입력 핀 ─ 스위치 ─ GND
```

스위치를 누르면 입력 핀이 GND와 연결되고, ZMK가 이를 키 입력으로 인식한다.

## 절대 하지 않을 것

```text
스위치를 3V3, 5V, VUSB에 연결하지 않는다.
```

| 핀 | 초기 테스트 사용 여부 | 이유 |
| --- | --- | --- |
| GND | 사용 | 스위치 공통 기준점 |
| D0~D3 | 사용 | 키 입력 핀 |
| 3V3 | 사용하지 않음 | 전원 핀 |
| 5V / VUSB | 사용하지 않음 | USB 전원 핀 |
| RST | 사용하지 않음 | 리셋용 |
| NFC | 사용하지 않음 | 키 입력 테스트 제외 |

## 1키 테스트

펌웨어:

```text
easysplit_1key
```

매핑:

```text
D0 → A
```

배선:

```text
D0 ─ 스위치 ─ GND
```

성공 기준:

```text
스위치를 누르면 A가 입력된다.
```

## 4키 테스트

펌웨어:

```text
easysplit_4key
```

매핑:

```text
D0 → A
D1 → B
D2 → C
D3 → D
```

배선:

```text
D0 ─ 스위치 ─ GND
D1 ─ 스위치 ─ GND
D2 ─ 스위치 ─ GND
D3 ─ 스위치 ─ GND
```

성공 기준:

```text
각 스위치를 누르면 A, B, C, D가 각각 입력된다.
```

## 브레드보드 주의사항

택트 스위치는 다리가 4개여도 내부적으로는 양쪽 두 다리씩 연결되어 있다.

주의할 점:

```text
스위치를 브레드보드 홈을 가로질러 꽂아야 한다.
같은 줄에 꽂으면 항상 연결된 상태가 될 수 있다.
```

## 부품 도착 후 첫 배선 순서

1. XIAO BLE의 USB-C 포트 방향을 기준으로 핀맵 이미지와 실제 보드를 맞춘다.
2. GND 위치를 찾는다.
3. D0 위치를 찾는다.
4. 1키 테스트는 D0와 GND만 연결한다.
5. 1키 성공 후 D1, D2, D3로 확장한다.

## 실패 시 확인

### 아무 입력도 안 됨

```text
- 블루투스 페어링 확인
- 펌웨어가 올바른지 확인
- D0와 GND 위치 확인
- 스위치 방향 확인
- 브레드보드 연결 방향 확인
```

### 일부 키만 안 됨

```text
- 해당 점퍼선 교체
- 해당 스위치 교체
- 해당 핀 위치 재확인
```

### 계속 입력되는 것처럼 보임

```text
- 스위치 다리 방향 오류 가능성
- 브레드보드 같은 라인에 잘못 꽂았을 가능성
- 입력 핀과 GND가 상시 연결되었을 가능성
```
